React Native 环境问题解决指南

解决 'react-native' 不是内部或外部命令的问题

遇到的问题

当运行 npx react-native run-android 时，系统提示：

'react-native' 不是内部或外部命令，也不是可运行的程序或批处理文件。

解决方案步骤

请按照以下步骤检查和修复您的React Native开发环境：

1

确认项目目录

确保您在正确的React Native项目目录中操作，该目录应包含：

  • package.json 文件
  • node_modules 文件夹
  • android 和 ios 文件夹
2

安装项目依赖

如果尚未安装依赖，请在项目根目录运行：

npm install

或使用Yarn（如果已安装）：

yarn install
3

使用npx运行命令

React Native推荐通过npx调用本地安装的CLI工具：

npx react-native run-android

这确保使用项目内部的react-native版本。

4

检查Node.js版本

确保使用兼容的Node.js版本（React Native 0.64+需要Node 12+）：

node --version

如果版本不兼容，请从Node.js官网下载安装LTS版本。

5

清理缓存并重新安装

有时缓存可能导致问题，尝试清理后重新安装：

npm cache clean --force
rm -rf node_modules
npm install
6

检查Android环境

确保Android开发环境配置正确：

  • JDK 8或11已安装
  • ANDROID_HOME环境变量已设置
  • Android SDK工具已安装

环境检查清单

# 诊断命令 - 在项目根目录运行这些命令进行检查

node --version # 检查Node.js版本

npm --version # 检查npm版本

npx react-native doctor # 检查React Native环境问题

npx react-native info # 显示环境信息

成功运行后

当环境配置正确后，运行以下命令应该能够成功启动Android应用：

npx react-native run-android

如果仍有问题，请尝试运行以下命令启动Metro打包器：

npx react-native start
